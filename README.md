# Tower Defense (Unity WebGL)

## Опис проєкту
**Tower Defense** — це 2D стратегічна гра жанру **Tower Defense**, створена в середовищі **Unity** мовою програмування **C#**.

Гравець виконує роль захисника бази та повинен зупинити хвилі ворогів, використовуючи різні типи оборонних веж.  
Гра підтримує **WebGL**, тому може запускатися безпосередньо у браузері.

---

# Геймплей
Основна мета гравця — захистити свою базу від хвиль ворогів.

Гравець:
- отримує стартове золото;
- купує оборонні вежі;
- розміщує їх на ігровому полі;
- отримує золото за знищення ворогів;
- повинен пережити всі хвилі атак.

Програш:
- якщо HP бази стає 0.

Перемога:
- якщо всі хвилі успішно пройдено.

---

# Реалізований функціонал

## Оборонні вежі
У грі доступно 4 типи веж:

### Archer (Лучник)
- середня швидкість атаки;
- атака по одній цілі;
- середній радіус дії.

### Freezer
- уповільнює ворогів;
- середня швидкість атаки;
- Ghost не піддається уповільненню.

### Mage
- магічна атака;
- атака по площі (AoE);
- короткий радіус дії.

### Cannon
- повільна атака;
- висока шкода;
- атака по площі.

---

## Вороги
Реалізовано 3 типи ворогів:

### Goblin
- швидкий;
- низький запас здоров’я;
- дешевий для спавну.

### Orc
- повільний;
- великий запас здоров’я;
- складніший для знищення.

### Ghost
- середня швидкість;
- не піддається ефекту уповільнення;
- особливий тип ворога.

---

# Ігрове поле
Гра використовує **Tilemap-сітку**.

Особливості:
- вороги рухаються по заздалегідь заданому маршруту (**waypoints**);
- вежі не можна ставити на дорогу;
- вежі можна ставити лише на вільні клітинки;
- вежі не можна ставити занадто близько одна до одної.

---

# Система хвиль
Реалізована повноцінна хвильова система:

- кожна хвиля генерується на основі бюджету;
- бюджет збільшується з кожною хвилею;
- вороги можуть змішуватися випадковим чином;
- максимум **50 ворогів на хвилю**;
- між хвилями є **фаза підготовки**;
- таймер автоматично запускає наступну хвилю;
- після завершення хвилі гравець отримує бонусне золото.

# Оптимізація
Для оптимізації гри реалізовано **Object Pooling**.

Використовується для:
- ворогів;
- снарядів.

Це дозволяє:
- уникнути постійного `Instantiate()` / `Destroy()`;
- зменшити навантаження на систему;
- покращити продуктивність у WebGL.

# Встановлення та запуск

Перед запуском потрібно встановити:

---

## Клонування репозиторію
Виконайте:

```bash
git clone https://github.com/RomaKiberline/TowerDefense.git
```

---

## Відкриття проєкту
1. Відкрити **Unity Hub**
2. Натиснути:

```text
Add Project
```

3. Обрати папку:

```text
TowerDefense
```

---

## Відкриття сцени
У Unity:

```text
Assets → Scenes → SampleScene
```

---

## Запуск гри
Натиснути кнопку:

```text
Play
```

---

# Запуск WebGL версії

## Збірка WebGL
У Unity:

```text
File → Build Profiles → Web → Build
```

або:

```text
File → Build Settings → WebGL → Build
```

---

## Локальний запуск
WebGL не запускається напряму через відкриття `index.html`.

Потрібно використати локальний сервер.

Перейти у папку збірки:

```bash
cd TowerDef
```

Запустити сервер:

```bash
python -m http.server 8000
```

Відкрити у браузері:

```text
http://localhost:8000
```
